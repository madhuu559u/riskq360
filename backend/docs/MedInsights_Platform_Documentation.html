Platform Overview

MedInsights is a medical chart extraction and analysis platform that processes PDF medical charts using LLM-based extraction and stores structured clinical data in PostgreSQL. It is designed for Risk Adjustment (HCC/RAF scoring) and HEDIS quality measure analysis.

What Problem Does This Solve?

Healthcare organizations need to extract structured clinical data from unstructured PDF medical charts for:

ProblemMedInsights Solution
Risk Adjustment — Payers need accurate HCC/RAF scores to predict member costs. Missing or incorrect codes = lost revenue (thousands per member). LLM extracts every ICD-10 code with negation awareness, maps to HCC categories, calculates RAF scores. Only "active" diagnoses count.
HEDIS Compliance — Health plans are rated on quality measures (BP control, cancer screenings, depression screening). Manual chart review is slow and expensive. Dedicated HEDIS pipeline extracts BP readings, lab results, screenings, eligibility conditions, and depression assessments with measure-specific targets.
Chart Abstraction — Clinical teams need structured encounter data, sentences, medications, procedures from narrative chart text. 5 parallel LLM pipelines extract demographics, sentences (22 categories), encounters with medications/procedures/labs/referrals.
Low-Quality PDFs — Scanned/handwritten charts have poor OCR quality. Quality scoring (0-100) per page with automatic GPT-4o vision fallback for pages below threshold (50).

Key Design Decisions

DecisionRationale
Dual Persistence (JSON + DB)JSON files are the primary source of truth. Database persistence is non-fatal — pipeline never crashes due to DB errors. JSON can be re-loaded later.
Quality-Based VisionGPT-4o vision is expensive ($0.01-0.03/page). Only triggered when page quality score < 50, saving costs on well-digitized charts.
Negation-Aware ExtractionOnly "active" diagnoses count for RAF scoring. Incorrect negation = incorrect RAF = financial impact (thousands per member).
Chunked + Deep MergeLong charts are split into ~10K char chunks processed in parallel. Results are merged: lists concatenate, dicts merge recursively.
ProcessingRun per Re-runEach chart can be processed multiple times (different models, settings). Each run creates a new ProcessingRun with full audit trail.
Review WorkflowClinical entities have review_status (pending/approved/rejected) for human-in-the-loop validation.

Extraction Pipeline

The pipeline processes a single PDF through 6 sequential steps, generating 8 JSON output files and persisting to the database.

Pipeline Steps

1

PDF Text Extraction — smart_pdf.py

Opens the PDF with PyMuPDF. For each page: extracts text, computes image area ratio, and scores text quality (0-100) using 12 heuristic factors. If score < quality_threshold (default 50), renders the page as a 200 DPI PNG and sends it to GPT-4o Vision for OCR extraction. Outputs full text with --- PAGE N --- markers.

Quality FactorDeductionWhat It Detects
Text length < 20 charsScore = length × 1.5Nearly empty pages
Text length < 50 chars-30Very sparse content
Character quality ratio < 0.70-40Garbled OCR output
Avg word length > 15-30Merged/garbled words
Consonant cluster ratio > 0.1-25Non-word gibberish (5+ consonants)
Avg line length < 5-20Fragmented extraction
Image area ratio > 0.7-25Image-heavy pages (scans)
2

LLM Extraction — openai_extractor.py

The full text is sent through 5 parallel pipelines, each with its own system prompt. Each pipeline:

  1. Chunks text at paragraph boundaries (~10,000 characters per chunk)
  2. Processes chunks in parallel (up to 4 concurrent API calls)
  3. Parses JSON response with robust 5-strategy fallback parser
  4. Deep merges results across chunks (lists concatenate, dicts merge)
PipelinePromptOutput FileWhat It Extracts
DemographicsPrompt 11_patient_demographics.jsonName, DOB, address, providers, vitals, social/family history
SentencesPrompt 22_clean_sentences.jsonClinical sentences with 22 categories + negation detection
Risk AdjustmentPrompt 33_risk_adjustment.jsonICD-10/ICD-9/SNOMED codes with 6 negation statuses
HEDISPrompt 44_hedis_evidence.jsonBP readings, labs, screenings, eligibility, depression
EncountersPrompt 55_encounters.jsonPer-visit data with meds, procedures, labs, referrals
3

Save Extraction JSONs

Each pipeline's results are saved as pretty-printed JSON files (indent=2) in results/<pdf_name>/. Files are numbered 1-5 corresponding to their pipeline.

4

HCC Post-Processing — orchestrator.process_hcc()

Maps ICD-10 codes to HCC V28 categories and calculates the total RAF score. Only "active" diagnoses count. Same HCC category from multiple codes is deduplicated (only first contributes to RAF). Saved as 6_risk_adjustment_hcc.json.

5

Summary Generation

Compiles metadata including PDF stats (pages, characters), extraction counts (sentences, diagnoses, encounters), HCC summary (RAF score, categories), processing time, and model info. Saved as 7_extraction_summary.json.

6

Database Persistence — PersistenceService

Non-fatal auto-persist to PostgreSQL. Creates Chart + ProcessingRun records, then reads all 7 JSON files and maps them to the 38 database tables. Commits in a single transaction. Failures log warnings but never crash the pipeline.

Robust JSON Parser

openai_extractor.safe_parse_json() uses 5 fallback strategies to handle messy LLM output:

#StrategyWhat It Handles
1Strip markdown fencesLLM wraps JSON in ```json ... ```
2Direct json.loads()Clean JSON output
3Remove trailing commasLLM adds commas before } or ]
4Walk backwards + add closersTruncated JSON (hit max_tokens limit)
5Regex extract {...} or [...]JSON embedded in prose text

API Call Structure

# Text extraction call (openai_extractor.py)
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user",   "content": f"Extract data from this medical chart text (chunk {idx+1}):\n\n{chunk}"}
    ],
    response_format={"type": "json_object"},
    max_tokens=4096,
    temperature=0.1
)

# Vision extraction call (smart_pdf.py)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": VISION_SYSTEM_PROMPT},
        {"role": "user",   "content": [
            {"type": "text", "text": "Extract all medical info..."},
            {"type": "image_url", "image_url": {
                "url": f"data:image/png;base64,{base64_image}",
                "detail": "high"
            }}
        ]}
    ],
    response_format={"type": "json_object"},
    max_tokens=4096
)

Key Persistence Details

ℹ
Single Transaction: All 7 files are persisted in one database transaction. A single commit() at the end ensures atomicity.
⚠
HCC Mapping Lookup: The most complex operation. Queries all Diagnosis rows for this chart/run, builds a lookup by (icd10_code, description.upper()), then matches each HCC mapping to its parent Diagnosis to set the foreign key.
✅
Non-Fatal: Database persistence failures log warnings but never crash the pipeline. JSON files are always the primary output.

HCC / RAF Scoring

Hierarchical Condition Category (HCC) mapping and Risk Adjustment Factor (RAF) score calculation. This is the core financial output of the platform.

What is HCC/RAF?

ℹ
HCC (Hierarchical Condition Categories) groups ICD-10 diagnosis codes into ~130 condition categories used by CMS (Medicare) to predict healthcare costs.
RAF (Risk Adjustment Factor) is a numeric score derived from HCC categories. Higher RAF = higher predicted cost = higher capitation payment from CMS. Each HCC category has a fixed RAF weight (e.g., HCC 85 Heart Failure = 0.368).

How MedInsights Calculates RAF

1

Extract Diagnoses (Prompt 3)

LLM extracts all ICD-10 codes from the chart with negation status (active/negated/resolved/historical/family_history/uncertain).

2

Filter: Only Active Diagnoses

Non-active diagnoses get hcc_category=null, raf_weight=0.0, counts_for_raf=false.

3

Map ICD-10 → HCC Category

Look up each active ICD-10 code in the ICD10_TO_HCC mapping table (20 codes). If not found: "No HCC mapping". If in NON_HCC_CODES: "Non-HCC code".

4

Deduplicate HCC Categories

If the same HCC category appears from multiple ICD-10 codes, only the first one contributes to RAF. Duplicates get counts_for_raf=false, reason="Duplicate HCC".

5

Sum RAF Score

Total RAF = sum of unique HCC category RAF weights, rounded to 3 decimal places.

Negation Detection

Negation detection is mission-critical in MedInsights. Incorrect negation classification directly impacts RAF scoring (financial) and HEDIS eligibility (quality ratings).

Two-Level Negation System

LevelWhereFieldValuesPurpose
Sentence Level Prompt 2 (Clean Sentences) is_negated (boolean) true / false Quick flag for clinical review
Diagnosis Level Prompt 3 (Risk Adjustment) negation_status (string) active / negated / resolved / historical / family_history / uncertain Determines RAF eligibility

Negation Detection Patterns

Pattern TypeTrigger WordsExampleResult
Direct Negationno, not, none, never, without, absent, deny"No chest pain"is_negated: true
Patient Denialdenies, denied, does not report/endorse"Patient denies SOB"is_negated: true
Test Negativenegative, non-reactive, unremarkable, WNL"Chest X-ray unremarkable"is_negated: true
Exclusionruled out, r/o, no evidence of, no signs of"PE ruled out"is_negated: true
Resolutionresolved, improved, cleared, remission"UTI resolved"is_negated: true

Critical Edge Cases

Chart TextCorrect ClassificationWhy
"Depression on remission with medication"activeStill being managed/monitored with medication
"Diabetes well controlled"active"Well controlled" doesn't mean resolved
"DM well controlled on metformin"activeActive treatment = active condition
"No anemia because she takes iron"resolvedCondition managed to resolution
"Resolved UTI"resolvedExplicitly stated as resolved
"Family history of breast cancer"family_historyNot patient's condition; Z-codes only
"Possible hypothyroidism"uncertainNot confirmed diagnosis
"NEGATIVE DEPRESSION SCREEN"negatedScreen was negative = no condition found
💰
Financial Impact: Each missed active HCC diagnosis can cost $500-$5,000+ in lost RAF revenue per member per year. Each incorrectly-active diagnosis can lead to audit penalties. Negation accuracy is directly tied to revenue integrity.
