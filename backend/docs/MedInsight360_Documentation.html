Unified Risk Adjustment & Quality Intelligence Platform

MedInsight 360 processes medical chart PDFs through 5 parallel AI extraction pipelines (LangExtract), then applies a 4-Track Ensemble HCC Engine (LLM extraction + TF-IDF classifier + BioClinicalBERT + LLM verification gate) for maximum diagnostic capture, maps to CMS-HCC V28 risk codes, evaluates 122 HEDIS quality measures via a rule-based YAML-driven engine, and generates audit-ready output packs with full evidence trails.

Core Objectives

  • ●5-Pipeline LangExtract — Demographics, Clinical Sentences (22 categories, 6-status negation), Risk Diagnoses (ICD-10), HEDIS Evidence (BP/labs/screenings/immunizations), Encounters — all run in parallel per text chunk with structured JSON output.
  • ●4-Track Ensemble HCC Engine — Combine LLM-extracted ICD codes, TF-IDF multi-label classifier (115 HCCs), trained BioClinicalBERT deep learning predictions, and GPT-4o verification gate. Apply V28 hierarchy suppression (21 groups) and compute RAF scores.
  • ●122-Measure HEDIS Engine — Rule-based engine with YAML-defined measure specifications, 168 value set JSONs, and a normalized MemberEventStore. Evaluates eligibility, denominator, exclusions, and numerator with full audit traces.
  • ●Audit-Ready Output — Generate complete evidence packs with page references, character offsets, negation status, MEAT validation, and provenance trails for every clinical finding.

LangExtract — 5 Parallel Pipelines

All 5 extraction pipelines run simultaneously on chunked clinical text. Each pipeline has a dedicated LLM prompt template, structured JSON output schema, and chunk-level merge/dedup logic.

P1

Demographics Pipeline

demographics_pipeline.py

Extracts: patient name, DOB, gender, MBI/member ID, insurance, address, phone, race/ethnicity, providers. Merges across chunks with conflict resolution.

P2

Clinical Sentences Pipeline

sentence_pipeline.py

Categorizes every clinical sentence into 22 categories (diagnosis, medication, vital, lab, allergy, etc.) with 6-status negation model: active, negated, resolved, historical, family_history, uncertain. Preserves source page + character offsets.

active negated resolved historical family_hx uncertain
P3

Risk Diagnosis Pipeline

risk_dx_pipeline.py

Extracts all mentioned diagnoses with: ICD-10 code, description, negation status, supporting text, date of service, provider, source section. This feeds directly into Track 1 of the HCC Ensemble.

P4

HEDIS Evidence Pipeline

hedis_pipeline.py

Extracts evidence for quality measures: BP readings (systolic/diastolic), A1C values, eye exams, colonoscopies, mammograms, medications, screenings, immunizations, depression screenings, falls risk assessments. Feeds into the HEDIS Engine (Step 6).

P5

Encounter Pipeline

encounter_pipeline.py

Extracts all encounters with: date of service, provider name, facility, visit type, CPT codes, diagnoses, medications ordered, labs ordered, referrals, procedures. Builds encounter timeline.

Parallel Execution Architecture

# extraction/parallel_extractor.py

Full PDF Text → chunk into N segments (default ~10K chars with overlap)

For each chunk:
  ThreadPoolExecutor(max_workers=5):
    ├ Thread 1: demographics_pipeline(chunk) → JSON
    ├ Thread 2: sentence_pipeline(chunk)      → JSON
    ├ Thread 3: risk_dx_pipeline(chunk)        → JSON
    ├ Thread 4: hedis_pipeline(chunk)           → JSON
    └ Thread 5: encounter_pipeline(chunk)       → JSON

Merge chunk results:
  → demographics: last-writer-wins for each field
  → sentences: append all, dedup by (page, char_start)
  → diagnoses: dedup by (icd10_code, supporting_text)
  → hedis: dedup by (type, date, value)
  → encounters: dedup by (date, provider)

4-Track Ensemble HCC Engine V4

Multi-strategy HCC extraction combining LLM, TF-IDF, BioClinicalBERT, and LLM verification gate. Catches 2-3x more payable HCCs than LLM alone with high precision through the verification gate.

Track 1: LLM Extraction (ICD → HCC)

Pipeline 3 (Risk Dx) extracts diagnoses with ICD-10 codes from clinical text. Active diagnoses are mapped to HCC categories via the V28 crosswalk table (7,903 mappings).

  • • Handles dotted (E11.65) and undotted (E1165) ICD formats
  • • Only active diagnoses (negated/resolved/family_history excluded)
  • • Groups multiple ICDs per HCC, carries supporting text
  • • Default confidence: 0.9
Input:  risk_data["diagnoses"] from Pipeline 3
Filter: negation_status == "active" only

For each active diagnosis:
  icd10 = "E11.65"  → V28 lookup
  → HCC37 (Diabetes with Chronic Complications)
  → raf_weight: 0.302

Typical yield: 1-8 HCCs per chart

Track 2: TF-IDF Multi-Label Classifier

Pre-trained sklearn OneVsRestClassifier with TF-IDF vectorizer. Trained on clinical text → 115 HCC labels. Catches patterns in long documents that LLM may miss.

  • • Vectorizer: 1.8MB TF-IDF model
  • • Classifier: 46MB OneVsRest model
  • • Thresholds: Per-label optimal (0.34–0.97)
  • • New HCCs not already found by Track 1 are added
Files:
  ml_engine/models/tfidf_vectorizer.pkl  (1.8MB)
  ml_engine/models/tfidf_classifier.pkl  (46MB)
  ml_engine/models/tfidf_thresholds.npy  (115 floats)

Process:
  text → TF-IDF vectorize → predict_proba
  → apply per-label thresholds
  → yield HCC codes above threshold

Track 3: BioClinicalBERT (Trained HCC Predictor)

Fine-tuned emilyalsentzer/Bio_ClinicalBERT with 115 sigmoid output heads. Trained on clinical notes for HCC prediction. The biggest contributor of novel HCC candidates.

  • • Model: 1.3GB best_model.pt
  • • Thresholds: Per-label optimal (0.09–0.94)
  • • Labels: 115 HCC categories (label_mapping.json)
  • • Text truncated to 512 tokens (BERT max)
Files:
  ml_engine/models/bioclinicalbert/
    model/best_model.pt         (1.3GB)
    model/optimal_thresholds.npy (115 floats)
    label_mapping.json           (115 HCC labels)

Process:
  text → tokenize (512 max) → BERT forward
  → 115 sigmoid outputs → apply thresholds
  → yield HCC codes above threshold

Typical yield: 3-6 predictions per chart
(Often finds HCCs that LLM missed)

Track 4: LLM Verification Gate

Every candidate HCC from Tracks 1-3 is verified by GPT-4o-mini with clinical evidence. The LLM confirms whether the diagnosis is actually documented and actively managed.

  • • Structured JSON response: {is_verified, confidence, reasoning}
  • • Minimum confidence: 0.3
  • • Evidence limited to 2K chars per candidate
  • • Typical rejection rate: 40-60% (filters ML false positives)
For each candidate HCC:
  Send to GPT-4o-mini:
    "Diagnosis Claimed: {hcc_description}
     Clinical Evidence: {relevant_text}
     Is this diagnosis PRESENT + MANAGED?"

  Response: {
    "is_verified": true/false,
    "confidence": 0.0-1.0,
    "reasoning": "brief explanation"
  }

  if verified AND confidence >= 0.3:
    → KEEP (payable)
  else:
    → REJECT (with reason logged)

Pipeline Integration Flow

# scripts/process_charts.py — Step 6

def _run_hedis_engine(extraction_results, assertions):
    # 1. Build adapter input from all 5 pipeline outputs
    adapter_data = {
        "demographics": extraction_results["demographics"],  # DOB, gender, age
        "assertions": assertions,                             # All clinical assertions
        "diagnoses": risk_data["diagnoses"],                 # ICD-10 codes from P3
        "encounters": encounters_data["encounters"],         # Visit dates from P5
        "labs": hedis_ext["lab_results"],                    # A1C, LDL from P4
        "vitals": hedis_ext["blood_pressure_readings"],      # BP readings from P4
        "medications": hedis_ext["medications_for_measures"], # Meds from P4
        "immunizations": hedis_ext["immunizations"],         # Vaccines from P4
    }

    # 2. Convert to MemberEventStore via adapter
    store = adapt_assertions(adapter_data)
    #    → DiagnosisEvent, ProcedureEvent, LabEvent, VitalEvent
    #    → MedicationEvent, EncounterEvent, ImmunizationEvent

    # 3. Run all 122 YAML measures
    engine = HedisEngine(measurement_year=2026)
    results = engine.evaluate_member(store)
    #    For each measure:
    #      1. Check eligibility (age, gender, required conditions)
    #      2. Check denominator inclusion
    #      3. Check exclusions (hospice, pregnancy, etc.)
    #      4. Check numerator compliance (required labs/procedures/values)
    #      5. Compute confidence score
    #      6. Generate gap details if not met

    # 4. Return: 122 measure results with traces + gap list

Scripts & CLI Tools

Command-line tools for processing charts, batch operations, and system setup.

scripts/process_charts.py Primary CLI

End-to-end chart processing: PDF → LangExtract (5 pipelines) → HCC Ensemble V4 → HEDIS Engine (122 measures) → DB persistence.

# Process all PDFs in default uploads/ directory
python scripts/process_charts.py

# Single PDF
python scripts/process_charts.py --pdf uploads/chart.pdf

# Batch process from custom directory
python scripts/process_charts.py --input-dir /path/to/pdfs --max-files 10

# Skip database persistence
python scripts/process_charts.py --no-db

# Custom API key and model
python scripts/process_charts.py --api-key sk-proj-... --model gpt-4o
