MedInsight 360

Functional Documentation — Platform Capabilities, Screens & Workflows

v1.0 · March 2026 · Production

Platform Objective

MedInsight 360 is a unified AI-powered platform that automates the analysis of medical chart PDFs to produce Payable HCC Packs (Risk Adjustment) and HEDIS Quality Gap Reports (Quality Intelligence) — reducing manual coder effort by 80% while maintaining audit-grade accuracy.

The Problem It Solves

Manual Chart Coding is Slow

A human coder takes 30-60 minutes per chart to identify HCC codes, validate ICD accuracy, check MEAT evidence, and evaluate HEDIS measures. With thousands of charts per year, this is unsustainable.

Missed Revenue & Quality Gaps

Human coders miss diagnoses buried in long documents. Missed HCCs = lost revenue. Missed HEDIS gaps = lower quality scores, failed audits, and member harm.

No Unified Platform

Risk Adjustment and HEDIS Quality are typically handled by separate teams with separate tools. MedInsight 360 unifies both into one workflow on one chart.

Core Value Propositions

  1. Chart-to-Code in 45 seconds — Upload a PDF, get a complete Payable HCC Pack with RAF score, MEAT evidence, and audit trail
  2. Evidence-Linked Everything — Every diagnosis, every HCC, every HEDIS finding links back to the exact text and page in the PDF
  3. 3-Source HCC Ensemble — Catches diagnoses that LLMs miss by combining ML predictions + ICD retrieval + LLM extraction
  4. Full Audit Trail — Every AI decision is traceable; every human review is logged; every change is audited
  5. Configurable Without Code — Swap LLM providers, adjust confidence thresholds, toggle features, edit prompts — all from the UI

Review Actions in Detail

Accepting a Finding

  1. Coder opens chart in Chart Viewer
  2. Navigates to HCC Pack tab or Diagnoses tab
  3. Reviews the AI-generated finding (reads evidence, checks PDF)
  4. Clicks "Accept" button (green checkmark)
  5. Review Modal opens with optional notes field
  6. Coder adds notes (optional) and clicks "Confirm"
  7. Status changes to ACCEPTED
  8. Audit log entry created: who, when, what

Rejecting a Finding

  1. Coder reviews finding and determines it's incorrect
  2. Clicks "Reject" button (red X)
  3. Review Modal opens with required reason field
  4. Coder enters reason (e.g., "Diagnosis is negated in context", "ICD specificity incorrect")
  5. Clicks "Confirm"
  6. Status changes to REJECTED
  7. Audit log entry created with rejection reason

Adding a Missing Diagnosis

  1. Coder notices a diagnosis the AI missed
  2. Option A: Click "Add Diagnosis" button in Diagnoses tab
  3. Option B: Select text in PDF → click stethoscope button
  4. Add Diagnosis Modal opens with fields:
    • ICD-10 code input with live BM25 search (type "diabet" → see matching codes)
    • Description (auto-filled from ICD lookup)
    • Supporting text (pre-filled if from PDF selection)
    • Page number
    • Negation status dropdown
    • Optional: date of service, provider
  5. Click "Save" → diagnosis added to chart
  6. Marked as CODER-ADDED in the system

ICD-10 Coding Helper

When adding or editing a diagnosis, the Coding Helper provides real-time ICD-10 suggestions:

  • Type 2+ characters in the ICD search field
  • BM25 ranking searches across 7,903 ICD-10-CM codes
  • Results show: code, description, HCC mapping (if any), RAF weight
  • Click a result to auto-fill the code and description
  • "Payment-only" filter toggle to show only HCC-mapped codes

Workflow 1: Process a New Chart

What happens behind the scenes:

  1. PDF saved to server, chart record created
  2. Orchestrator: PDF text extraction → quality scoring → OCR if needed
  3. Text chunked and sent to LLM (MiniMax single-pass)
  4. 500+ assertions extracted with codes, dates, quotes, negation
  5. HCC ensemble: LLM + ML + TF-IDF candidates merged
  6. NegEx filters, GPT-4o verifies, V28 maps, hierarchy applies
  7. HEDIS engine evaluates 100+ measures against assertions
  8. All results persisted to database
  9. Frontend fetches and displays results

Workflow 2: Review & Code a Chart

  1. Open chart from Chart List (click card)
  2. Start with HCC Pack tab:
    • Review RAF score — is it reasonable?
    • For each payable HCC: click to expand, review supporting ICDs
    • Check MEAT evidence — is M/E/A/T documented?
    • Click "View Evidence" to see supporting text highlighted in PDF
    • Accept or Reject each HCC
  3. Switch to Diagnoses tab:
    • Review all extracted diagnoses
    • Check negation status — is "active" correct?
    • Verify ICD code specificity (E11.65 vs E11.9?)
    • Accept, Reject, or Edit each diagnosis
    • Add any missed diagnoses using ICD lookup
  4. Switch to Care Gaps tab:
    • Review HEDIS measures marked as MET — verify evidence
    • Review measures marked as GAP — are they true gaps?
    • Accept or Reject each measure status
  5. Save Document: Click "Save" to persist all review decisions

Workflow 3: Find a Specific Diagnosis

  1. Open chart in Chart Viewer
  2. Use PDF search (Ctrl+F) to search for clinical term (e.g., "diabetes")
  3. See all matches highlighted in yellow across all pages
  4. Click a match in the dropdown to jump to that page
  5. Or: use Diagnoses tab search/filter to find specific ICD codes
  6. Click "View Evidence" on any diagnosis to highlight its source in the PDF

Workflow 4: Batch Processing

  1. Upload multiple charts via the Upload modal (one at a time, or use CLI for batch)
  2. Charts process in background (configurable parallelism: 3 concurrent charts)
  3. Chart List auto-refreshes every 10 seconds showing processing status
  4. Use Dashboard to monitor overall progress and success rates
  5. Filter Chart List by status "completed" to review finished charts

CLI batch mode:

python scripts/run_batch.py --input-dir ./charts/ --mode FULL --max-concurrent 3

Workflow 5: Configure the System

  1. Navigate to Settings (gear icon or /settings)
  2. Switch LLM provider: AI Engine tab → change provider dropdown → Save
  3. Disable HEDIS: Feature Flags tab → toggle HEDIS off → Save
  4. Adjust sensitivity: ML & Risk tab → lower ML threshold for more candidates → Save
  5. Edit prompts: Prompt Templates tab → modify extraction prompt → Save
  6. Changes take effect on next chart processing (no restart needed)

Workflow 6: Investigate a Specific HCC

  1. Open chart in Chart Viewer
  2. Go to HCC Pack tab
  3. Find the HCC (e.g., HCC 18 — Diabetes with Chronic Complications)
  4. Click to expand → see supporting ICD codes
  5. For each ICD: read confidence score, check ML vs LLM agreement
  6. Check MEAT evidence — all 4 elements present?
  7. Click "View Evidence" on the ICD to jump to PDF location
  8. Read the highlighted text in context
  9. If correct: Accept HCC. If wrong: Reject with reason.
  10. Check Suppressed section to see if any related HCCs were hierarchy-suppressed
