System Overview

What It Does

The HEDIS Engine is a production-grade, rule-based measure evaluation system that answers four questions for each member:

  1. 1Is this measure applicable (eligible) for the member?
  2. 2Is the member compliant? If not, what gap remains?
  3. 3What evidence supports compliance (dates + references)?
  4. 4What data is missing that prevents determination?

Key Design Principles

  • ✓Deterministic — No ML for eligibility/compliance. Pure rule-based evaluation
  • ✓Auditable — Every decision returns a structured trace with evidence
  • ✓Extensible — YAML-driven definitions; add measures without code changes
  • ✓Lenient — Handles partial data gracefully; never crashes on missing fields
  • ✓Separated — Measure logic decoupled from code sets (value set abstraction)
  • ✓6-Status Polarity — Active, Negated, Resolved, Historical, Family History, Uncertain

5-Step Evaluation Pipeline

👤
1. Eligibility
Age + Gender + Enrollment
_check_eligibility()
📋
2. Denominator
Diagnosis + Procedure + Encounter + Medication
_check_denominator()
🚫
3. Exclusions
Hospice, Pregnancy, ESRD, etc.
_check_exclusions()
✅
4. Numerator
all_of / any_of criteria + Inverse logic
_check_numerator()
🎯
5. Confidence
Deterministic score 0.0–1.0
_compute_confidence()

Code Reference

YAML Measure Schema

id: "CDC-A1C-TEST"
name: "Diabetes Care - HbA1c Testing"
version: "2025"
domain: "diabetes"
description: "Percentage of members 18-75 with diabetes who had an HbA1c test"

# Optional: for overuse/safety measures where numerator = bad event
inverse: true

eligibility:
  age: {min: 18, max: 75, as_of: "12-31"}
  gender: ["male", "female"]
  continuous_enrollment: true

denominator:
  requires_diagnosis:              # Primary condition
    valueset: "VS_DIABETES_ICD10"
    lookback_months: 24
    status: ["active", "historical"]
  requires_diagnosis_secondary:    # Co-morbidity (SMC, SMD)
    valueset: "VS_CVD_ICD10"
  requires_procedure:              # Procedure-based denominator
    valueset: "VS_DELIVERY_CPT"
  requires_encounter:              # Visit-based denominator
    encounter_types: ["emergency"]
  requires_medication:             # Drug-based denominator (AMR)
    medication_class: "asthma"

exclusions:
  - type: "diagnosis"
    valueset: "VS_HOSPICE"
    lookback_months: 12

numerator:
  all_of:                          # AND logic (all must be met)
    - type: "lab_exists"
      lab: "A1C"
      window_months: 12
  any_of:                          # OR logic (any one suffices)
    - type: "procedure"
      valueset: "VS_COLONOSCOPY_CPT"

valuesets_needed:
  - VS_DIABETES_ICD10
  - VS_HOSPICE

Engine Usage Example

from hedis_engine.engine import HedisEngine
from hedis_engine.types import *
from datetime import date

# Initialize engine
engine = HedisEngine(measurement_year=2024)
print(f"Loaded {len(engine.measures)} measures")

# Build member data
store = MemberEventStore(
    demographics=Demographics(dob=date(1970, 1, 1), gender=Gender.MALE),
    diagnoses=[
        DiagnosisEvent(code="E11.9", code_system=CodeSystem.ICD10CM,
                       status=DiagnosisStatus.ACTIVE, event_date=date(2024, 3, 1)),
    ],
    labs=[
        LabEvent(test_type="A1C", value=7.5, unit="%", event_date=date(2024, 6, 1)),
    ],
)

# Evaluate all measures
results = engine.evaluate_member(store)
for m in results.measures:
    if m.applicable:
        print(f"{m.measure_id}: {m.status.value}")

# Evaluate specific measures
results = engine.evaluate_member(store, measure_ids=["CDC-A1C-TEST", "CBP"])

# Export to JSON
import json
print(json.dumps(results.to_dict(), indent=2))

def test_cdc_a1c_met():

    store = _store(
        demographics=Demographics(dob=date(1970, 1, 1), gender=Gender.MALE),

Command-Line Usage

# Evaluate all measures for a member
python -m hedis_engine --input ./member.json --year 2024 --out ./results.json

# Evaluate specific measures only
python -m hedis_engine --input ./member.json --year 2024 --measures CDC-A1C-TEST,CBP,BCS

# Print human-readable summary
python -m hedis_engine --input ./member.json --year 2024 --summary

# Custom catalog directory
python -m hedis_engine --input ./member.json --catalog ./my_measures/ --year 2025

# PDF evaluation (all 4 test charts)
